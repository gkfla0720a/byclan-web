## 🏗️ 시스템 구조도

### 핵심 3가지 요소

```
┌────────────────────────────────────────────────────────────────┐
│                      ByClan 앱 (웹사이트)                         │
│  사용자가 실제로 보고 클릭하는 화면과 버튼들                      │
└──────────────────────┬─────────────────────────────────────────┘
                       │
                       │ (데이터를 주고받음)
                       ↓
┌────────────────────────────────────────────────────────────────┐
│               Supabase 인증 서비스 (로그인 담당)                  │
│  • 비밀번호 저장·확인 ✓                                         │
│  • 로그인 토큰 발급 ✓                                           │
│  • Discord/Google 연동 ✓                                       │
└──────────────────────┬─────────────────────────────────────────┘
                       │
                       │ (사용자 정보 저장)
                       ↓
┌────────────────────────────────────────────────────────────────┐
│               PostgreSQL 데이터베이스 (정보 저장소)              │
│  • profiles 테이블: 닉네임, 역할, 포인트 등                      │
│  • profile_oauth 테이블: Discord/Google 연동 정보                │
│  • ladder_rankings 테이블: 게임 전적                            │
│  • profile_meta 테이블: 추가 설정 정보                           │
└────────────────────────────────────────────────────────────────┘
```

## 🔐 로그인 프로세스

### 상황 1️⃣: 자체 아이디(일반 계정)로 로그인

**사용자 입장에서의 단계:**

```
① 로그인 페이지 방문
   ↓
② 아이디 입력 (예: "myname")
   ↓
③ 비밀번호 입력
   ↓
④ "LOGIN SYSTEM" 버튼 클릭
   ↓
⑤ 로그인 성공 → 대시보드로 이동 ✓
```

**내부에서 일어나는 일:**

```
사용자가 "myname" + 비밀번호 입력
         ↓
앱이 내부 이메일로 변환: "login.myname@auth.byclan.local"
         ↓
Supabase에 "이 이메일과 비밀번호로 로그인 가능한가?" 물어봄
         ↓
✓ 가능 → 로그인 토큰 받음 (=표지증 같은 것)
✗ 불가능 → "아이디나 비밀번호가 틀렸습니다" 메시지
         ↓
토큰을 브라우저의 "쿠키" 같은 곳에 저장
(앱에 접속할 때마다 이 토큰을 자동으로 보내기 때문에 다시 로그인할 필요 없음)
         ↓
DB에서 사용자 정보 가져오기:
  • 닉네임 (By_myname)
  • 역할 (신입/일반/관리자 등)
  • 보유 포인트
  • 게임 전적 등
         ↓
앱 화면 업데이트 → 로그인한 상태로 표시
```

**순서도:**

```
┌─────────────────────────┐
│ 1. 로그인 페이지 도착     │
└────────────┬────────────┘
             │
             ↓
┌─────────────────────────┐
│ 2. 아이디 입력           │
└────────────┬────────────┘
             │
             ↓
┌─────────────────────────┐
│ 3. 비밀번호 입력         │
└────────────┬────────────┘
             │
             ↓
┌─────────────────────────────────────┐
│ 4. Supabase에 확인 요청              │
│    (login.myname@auth.byclan.local)  │
└────────────┬────────────────────────┘
             │
      ┌──────┴──────┐
      ↓             ↓
   ✓ 맞음         ✗ 틀림
      │             │
      ↓             ↓
   로그인성공    에러메시지
   토큰 받음     표시

```

```
   (이미 Discord에 로그인되어 있으면 이 단계 건너뜀)

   ↓

         ↓
Supabase에 등록: "Discord 계정으로 로그인한 사람입니다"
         ↓
DB에 저장: Discord ID, 사용자명 등
         ↓
앱에 로그인 토큰 발급 → 로그인 완료
```

## 📝 회원가입 프로세스

### 전체 흐름

```
사용자 입장에서:

① 로그인 페이지에서 "No account yet? Register Here" 클릭
   ↓
② 회원가입 폼으로 변경됨
   ↓
③ 원하는 닉네임 입력 (예: "Champion")
   ↓
④ "중복확인" 버튼 클릭 (이 닉네임이 이미 사용중인지 확인)
   ↓
⑤ ✓ "사용 가능한 닉네임입니다!" 메시지
   (만약 이미 누가 쓰고 있으면 다른 닉네임 시도)
   ↓
⑥ 로그인에 사용할 아이디 입력
   (이 아이디는 로그인할 때만 쓰고, 
    닉네임(By_Champion)은 클랜 내에서 보이는 이름)
   ↓
⑦ 비밀번호 입력 (8글자 이상, 영문+숫자 필수)
   ↓
⑧ 비밀번호 다시 입력해서 맞는지 확인
   ↓
⑨ 약관 동의 체크박스에 체크 ✓
   ↓
⑩ "REGISTER ACCOUNT" 버튼 클릭
   ↓
⑪ ✓ 가입 완료! "로그인을 진행하세요" 메시지
   자동으로 로그인 폼으로 돌아감
   ↓
⑫ 가입한 아이디와 비밀번호로 로그인
```

```

         ↓

         ↓
Supabase에 새 계정 생성 요청:
  • 이메일: login.[아이디]@auth.byclan.local
  • 비밀번호: 입력받은 비밀번호
         ↓
✓ 계정 생성됨
```

## 🔗 소셜 연동 프로세스

### Discord 연동하기

사용자가 **프로필 설정 페이지**에서:

```
① 현재 계정으로 로그인된 상태 (예: 자체 아이디 계정)
   ↓
② "프로필 설정" → "소셜 계정 연동" 섹션으로 이동
   ↓
③ Discord 섹션에서 "연동하기" 버튼 클릭
   ↓
④ Discord 로그인 페이지로 이동 (필요시)
   ↓
⑤ "이 앱 허용" 클릭
   ↓
⑥ ByClan으로 돌아옴
   ↓
⑦ "Discord 계정이 성공적으로 연동되었습니다" 메시지
   ↓
⑧ 이제 Discord로도 로그인 가능 ✓
```

**내부 프로세스:**

```
사용자가 "연동하기" 클릭
         ↓
Supabase linkIdentity 함수 실행:
  "현재 로그인한 계정에 Discord 붙여주세요"
         ↓
Discord 공식 페이지로 이동
         ↓
사용자가 "허용" → Discord 정보 반환
         ↓
profile_oauth 테이블에 저장:
  • discord_id: Discord 고유 ID
  • 연동 완료 표시
         ↓
✓ 연동 완료
앱에서 "Discord로도 로그인 가능" 표시
```

### Discord 연동 해제하기

```
사용자가 프로필 → Discord 섹션 → "연동 해제" 클릭
         ↓
확인 대화창: "정말 연동을 해제할까요?
            Discord가 유일한 로그인 수단이면 로그아웃됩니다"
         ↓
사용자가 "확인" 클릭
         ↓
DB에서 Discord 정보 삭제
         ↓
Supabase에서 Discord identity 제거
         ↓
다른 로그인 수단 있는가?
  • ✓ 있음 → 연동만 해제, 로그인 유지
  • ✗ 없음 → 로그아웃 후 홈으로 이동
```

### Google 연동도 동일한 프로세스

Discord 연동과 완전히 같은 방식입니다. 단, 저장되는 정보는:
- google_sub (Google 고유 ID)
- google_email
- google_name
- google_avatar_url

## 🚪 로그아웃 및 계정 보안

### 로그아웃

```
① 프로필 페이지 또는 헤더 메뉴에서 "Logout" 버튼 클릭
   ↓
② 로그아웃 확인 대화
   ↓
③ 사용자가 "확인" 클릭
   ↓
④ 내부 프로세스:
   • Supabase 토큰 삭제
   • 브라우저 쿠키/로컬 스토리지 초기화
   • 앱의 로그인 정보 제거
   ↓
⑤ 홈 페이지로 이동 (로그인하지 않은 상태)
```

### 비밀번호 변경

```
① 프로필 설정 페이지 → "🔒 계정 보안" 섹션
   ↓
② 현재 비밀번호 입력
   ↓
③ 새 비밀번호 입력 (8글자+영문+숫자)
   ↓
④ 새 비밀번호 확인 입력
   ↓
⑤ "비밀번호 변경" 버튼 클릭
   ↓
⑥ 내부 프로세스:
   • 현재 비밀번호로 재인증 (확인: "이 사람 맞음!")
   • 확인 완료 → 새 비밀번호로 업데이트
   ↓
⑦ ✓ "비밀번호가 성공적으로 변경되었습니다"
```

### 이메일 변경 (계정당 고급 보안)

```
① 프로필 설정 페이지 → "계정 보안" 섹션
   ↓
② 새 이메일 주소 입력
   ↓
③ "이메일 변경" 클릭
   ↓
④ 시스템이 2가지 이메일로 확인 링크 발송:
   • 기존 이메일에 링크 1개
   • 새 이메일에 링크 1개
   ↓
⑤ 사용자가 기존 이메일에서 링크 클릭
   ↓
⑥ 사용자가 새 이메일에서 링크 클릭
   ↓
⑦ ✓ "이메일 변경 완료!"
   (왜 2번? 해킹 방지 - 정말 둘 다 본인 소유인지 확인하는 절차)
```

## 🎯 핵심 정리

### 📌 "로그인 토큰"이란?

```
쉽게 말해: 
  = 영화관 회원카드 같은 것
  = 매번 신분증을 보여줄 필요 없이, 카드 하나로 출입
  = 앱에 자동 저장되어 다음 방문 때 자동으로 인식
```

### 📌 "Supabase"와 "DB"의 관계

```
Supabase = 편의점 (로그인·인증 담당)
DB = 편의점 창고 (사용자 정보 저장)

둘이 함께 동작:
  ① 회원가입 → Supabase에서 계정 생성
  ② 동시에 → DB에 프로필 정보 저장
  ③ 로그인 → Supabase에서 확인
  ④ 앱 시작 → DB에서 프로필 로드
```

### 📌 비밀번호는 어디에?

```
❌ DB에는 비밀번호가 저장되지 않음!
✅ Supabase 인증 서버에만 암호화되어 저장됨

왜? 보안 때문
  • DB가 해킹되어도 비밀번호는 안전
  • Supabase는 보안 전문 회사가 관리
```

### 📌 로그인 방식 비교표

| 방식 | 아이디 | 비밀번호 | 가입 난이도 | 로그인 속도 |
|------|--------|---------|----------|---------|
| **자체 아이디** | 자기가 만듦 | 자기가 설정 | 조금 복잡 (닉네임 중복확인 필요) | 보통 |
| **이메일** | 자신의 이메일 | 자기가 설정 | 간단 | 보통 |
| **Discord** | Discord 계정 | Supabase 관리 | 가장 간단 | 가장 빠름 (클릭 몇 번) |
| **Google** | Google 계정 | Supabase 관리 | 가장 간단 | 가장 빠름 (클릭 몇 번) |

## 🔍 문제 해결 가이드

### Q: 로그인이 안 돼요
```
✓ 확인사항:
  1. 아이디/이메일 맞나요?
  2. 비밀번호 맞나요? (대소문자 구분 주의)
  3. 인터넷 연결 확인
  4. 브라우저 쿠키 활성화되어 있나요?
```

### Q: 회원가입 중 "이미 사용중인 닉네임"이라고 나와요
```
✓ 해결 방법:
  • 다른 닉네임으로 시도
  • 예: "Protoss_Master" → "Protoss_Master2"
  • (By_는 자동으로 붙으므로 입력하지 않기)
```

### Q: Discord 연동이 안 돼요
```
✓ 확인사항:
  1. Discord 계정 만들었나요?
  2. 이미 다른 계정에 연동되지 않았나요?
     → 기존 계정에서 먼저 연동 해제 필요
  3. 인터넷 연결 확인
```

### Q: 비밀번호를 잊었어요
```
현재 버전: 재설정 기능 미구현
임시 해결: 클랜 관리자(마스터)에게 문의
```

## 🎓 요약

### 1️⃣ 회원가입
- 닉네임 정하고 중복확인
- 비밀번호 설정
- 약관 동의
- 완료!

### 2️⃣ 로그인
- 아이디/이메일 + 비밀번호 입력
- 또는 Discord/Google 클릭 (더 간단)
- 토큰 받아서 자동 저장
- 프로필 정보 로드

### 3️⃣ 로그인 유지
- 토큰이 자동으로 저장되어
- 다음 날 다시 방문해도 로그인 상태 유지
- (로그아웃하지 않은 한)

### 4️⃣ 소셜 연동
- Discord/Google 계정과 연결
- 나중에 소셜 계정으로도 로그인 가능
- 언제든 연동 해제 가능

### 5️⃣ 보안
- 비밀번호는 Supabase 전문 서버에만 저장
- DB에는 저장 안 됨
- 비밀번호 변경 시 현재 비밀번호로 재인증
- 이메일 변경 시 양쪽 확인 필요
