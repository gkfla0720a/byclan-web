### 로그인 성공했을 때: "토큰"을 받는다

```

{

    id: "a1b2c3d4-1234-5678-abcd-ef1234567890",    ← 사용자 고유 ID
    email: "login.junho@auth.byclan.local",        ← 로그인한 이메일
    user_metadata: {

    }

    refresh_token: "ey...",
    expires_in: 3600,                              ← 1시간 후 만료
    expires_at: 1234567890                         ← 언제 만료되는지
  }

```

### 로그인 후 앱에서 어떻게 인식할까?

```
📍 위치: src/hooks/useAuthSession.ts (라인 26)

supabase.auth.onAuthStateChange() 리스너 설정
  ↓
Supabase에서 "로그인 상태가 바뀌었어!" 신호를 받음
  ↓
event === 'SIGNED_IN' 인지 확인 (로그인됨)
  ↓
setUser(session?.user) → 사용자 정보 저장
  ↓
앱의 모든 컴포넌트가 "사용자가 로그인했구나" 인식!
```

**코드:**

```javascript
// src/hooks/useAuthSession.ts (라인 26-34)
const { data: { subscription } } = supabase.auth.onAuthStateChange((event, session) => {
  if (!isMounted) return;
  
  if (event === 'SIGNED_OUT') {
    setUser(null);  // 로그아웃 상태
  } else {
    setUser(session?.user || null);  // 로그인 상태
  }
  setSessionLoading(false);
});
```

## 🎯 핵심 정리

### 📌 최초 진입 시 일어나는 일

```
1️⃣ 앱 로드
   ↓
2️⃣ AuthProvider가 useAuth() 실행
   ↓
3️⃣ useAuthSession이 localStorage에서 토큰 확인
   ↓
4️⃣ 토큰 있으면? → 유효성 검사 → 사용자 정보 복구
   없으면? → 로그인 필요 상태로 표시
   ↓
5️⃣ useProfileData가 프로필 로드 (있으면)
   ↓
6️⃣ 모든 정보가 Context에 저장
   ↓
✅ 앱 준비 완료!
```

### 📌 토큰은 언제 갱신되는가?

```
autoRefreshToken: true 설정 덕분에:

1️⃣ 매 요청 전에 토큰 만료 시간 확인
   ↓
2️⃣ 만료 예정? → refresh_token으로 새 토큰 요청
   ↓
3️⃣ 새 토큰 받음 → localStorage 업데이트
   ↓
4️⃣ 사용자는 아무것도 모름! (자동 처리)
   ↓
5️⃣ 거의 로그아웃되지 않음!
```

### 📌 비밀번호는 절대 앱에서 저장하지 않는다

```
흐름:

비밀번호 입력
   ↓
HTTPS로 암호화
   ↓
Supabase 서버로 전송
   ↓
Supabase가 받아서 해시 후 자신의 DB에만 저장
   ↓
앱 서버나 일반 DB에는 저장 안 함!
   ↓
대신 토큰을 받아서 저장
   ↓
이후 모든 요청에 이 토큰만 사용
```
